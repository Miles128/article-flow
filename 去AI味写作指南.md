## 三、AI 句式库（10 大类句式识别与替换）

### 快速检测正则

```regex
(不是.{1,20}而是|既.{1,15}又|通过.{1,20}实现|一方面.{1,30}另一方面|不仅.{1,20}[还更甚至]|无论.{1,20}都|不管.{1,20}都|从.{1,15}到|让.{1,15}更|使.{1,15}更)
```

### 1. "不是...而是..."句式（≥2次必改）

```
❌ "这不是一个简单的工具，而是一个智能助手"
✅ "这是智能助手，不只是工具" 或直接陈述："重点是质量"
```

### 2. "既...又..."句式（≥2次必改）

```
❌ "既功能强大，又操作简单"
✅ 拆成两句："功能全。上手也简单，我10分钟就会用了"
```

### 3. "通过...实现..."句式（≥3次必改）

```
❌ "通过优化算法实现性能提升"
✅ 删除"通过"："优化算法后，性能提升了40%"
```

### 4. 连续三段式（≥3段必改）

```
❌ "第一...第二...第三..." 或 "在XX方面...在XX方面...在XX方面..."
✅ 打破对称：第一段用"首先"，第二段换"另外"，第三段用"最重要的是"
```

### 5. "一方面...另一方面..."句式（≥2次必改）

```
❌ "一方面提升了效率，另一方面降低了成本"
✅ "Claude Code让我快了40%。错误也少了，bug降了30%。"
```

### 6. "不仅...还..."句式（≥2次必改）

```
❌ "不仅速度快，还很准确"
✅ 用列表："有两个优点：快（2秒）+准（95%）"
```

### 7. "无论...都..."句式（≥1次必改，极易识别为AI）

```
❌ "无论是新手还是资深开发者，都能快速上手"
✅ "新手10分钟能上手。资深开发者更快，5分钟就会用了。"
```

### 8. "从...到..."句式（≥3次必改）

```
❌ "从安装到使用，全程只需5分钟"
✅ 拆解步骤："安装2分钟，配置1分钟，写第一段代码2分钟。总共5分钟。"
```

### 9. "让...更..."句式（≥4次必改）

```
❌ "让开发更高效，让代码更优雅"
✅ 说具体结果："开发快了40%。代码也更简洁，少了30%行数。"
```

### 10. 因果链句式（1句中≥3个因果词必改）

```
❌ "因为...所以...因此...从而..."
✅ 拆成多句："Claude Code理解上下文。所以代码准。调试少，效率高。"
```

## 五、审校流程

### 第一遍：快速扫描高危词

对照**第一章禁止清单**，扫一遍全文，出现以下必改：
- "本质上""最核心""至关重要""不难发现""显而易见"
- "综上所述""总而言之""值得注意的是""需要指出的是"
- "这标志着""这预示着""具有重要意义"

### 第二遍：套话扫描

对照**第二章**，检查每段开头和结尾：
- 开头有没有"在当今""随着XX发展"
- 结尾有没有"综上所述""总的来说"
- 程度副词有没有空洞的"显著""大幅度"
- "进行XX""实施XX"直接删掉"进行/实施"

### 第三遍：句式扫描

对照**第三章**，用正则查找：
```bash
grep -E "(不是.{1,20}而是|无论.{1,20}都|一方面.{1,30}另一方面)" draft.md
```
- "不是...而是..." ≥2 次改
- "无论...都..." 出現就改
- "一方面...另一方面..." ≥2 次改
- 连续三段相同句式改

### 第四遍：词汇替换

对照**第四章**，逐个替换书面词汇。

**目标量化**：
- 每 1000 字书面语不超过 5 个
- 每 1000 字 AI 句式不超过 2 个
- 每 1000 字套话不超过 3 个
- 最终 AI 检测率低于 30%
